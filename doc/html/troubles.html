Troubleshooting

If you don't find your answer here, which is likely, please email us at savonet-users. We are waiting for your questions, and will update this file quickly.

Parse error

The parser is not very smart and currently requires you to put many many parenthesis, around products and list elements. When it is lost, the error message is not very informative. Think about adding extra parenthesis, maybe you forgot some.

For example, we need you to write [ (op "foo") ; (op "bar") ] with the parenthesis. It is planned to work on that annoying stupidity.

Scheduler died !

Check the liveness of your scheduler. Maybe it just lacks a source that can always deliver frames. local_playlist, or one_file with a local file are perfect for that. Common error: maybe you want to play a local playlist but you use it via a playlist source ? Try local_playlist !

Unfortunately, this is also the most common serious bug in liquidsoap. So if you still don't understand why it happens, please contact me.

Strange source name

You may see in requests metadatas a field source set to something like (nobody.110). It should only happen if the request was created at the creation of the source, before that the parser gives a name to the source. I think it only happens for the infallible one_file, which happens when the URI is static. If you discover any other case, please tell me.

My scheduling intervals are misinterpreted

When you write 0-1h it means "from midnight until 2:00". It should be read as "as long as the hour digit is between 0 and 1".
